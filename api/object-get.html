Get Object ​

Retrieve a single device by its unique identifier.

Scope ​

Requires objects scope.

Request ​

http
GET /api/object

Authentication Required: Must include JWT token in Authorization header.

Request Body ​

Accepts a JSON object with the following shape:

json
{
  "uid": "AABBCCDDEEFF"
}

Note: The server accepts the request body as JSON (see examples).

Request Example ​

bash
curl -X GET "http://your-server-ip:port/api/object" \
     -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"uid":"AABBCCDDEEFF"}'

Response ​

Successful response uses the following JSON envelope:

json
{
  "success": true,
  "device": {
    "id": 123,
    "partial_uid": 456789,
    "oid": 987654,
    "uid": "AABBCCDDEEFF",
    "registered_at": "2025-10-18T12:34:56+02:00",
    "iccid": "1234567890123456789",
    "signal_level": 4,
    "com_type": 1,
    "con_type": 2,
    "ip": "192.168.1.100",
    "ping_interval": 60,
    "sms_ping_interval": 120,
    "hw_type": 1,
    "fw_version": "1.2.3",
    "receiver_id": 1,
    "receiver_nr": 2,
    "line_nr": 3,
    "last_activity": 1634567890,
    "status": 1,
    "last_sms_activity": 1634567891,
    "has_received_sms": true,
    "cloud_access_code": "XYZ123",

      "sessions": [],
      "average_session_time": 0
    },
    "related_objects": [],
    "module_id": 42,
    "last_introduction_received": 0
  }
}

When device is not found, the response will have device field set to null.

json
{
  "success": true,
  "device": null,
}

Response Fields ​

  • success (boolean): Whether the request succeeded.
  • device (object|null): When success is true, contains the device object; otherwise null.
  • error (string, optional): Error message when success is false.

Error Responses ​

Errors are reported as plain text or as response object defined above.

400 Bad request ​

Field error will contain some information on what went wrong.

json
{
  "success": false,
  "device": null,
  "error": "string"
}

401 Unauthorized ​

NOT_LOGGED_IN

403 Forbidden ​

FORBIDDEN

405 Method Not Allowed ​

Method Not Allowed

Usage Example (JavaScript) ​

javascript
async function getObject(uid) {
  const res = await fetch('/api/object', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ uid })
  });

  if (!res.ok) throw new Error(await res.text());
  const data = await res.json();
  if (!data.success) throw new Error(data.error || 'Unknown error');
  return data.device;
}
