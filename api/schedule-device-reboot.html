Schedule Device Reboot ​

Schedules a reboot for a device. The reboot is performed after the device connects to the server and sends its first message (or the next message if the device is already connected).

Request ​

http
POST /api/schedule-device-reboot

Authentication Required: Must include JWT token in Authorization header.

Required Scope: objects

Content-Type: application/json (JSON request body required)

Request Body ​

Parameters are passed as a JSON object. Example schema:

json
{
  "uid": "string",
  "cloud_access_code": "string"
}

Request Examples ​

Basic Request (curl) ​

bash
curl -X POST "http://your-server-ip:port/api/schedule-device-reboot" \
     -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"uid":"123456789012","cloud_access_code":"123456"}'

Using JavaScript ​

javascript
const response = await fetch('/api/schedule-device-reboot', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ uid: '123456789012', cloud_access_code: '123456' })
});

Response ​

Response JSON schema:

json
{
  "success": true,
  "error": "optional error message"
}

Success (200 OK) ​

json
{
  "success": true
}

Client Error (400 Bad Request) ​

Returns JSON with success: false and an error message. Possible error messages include:

  • "invalid request body"
  • "cloud_access_code must not be empty"
  • "uid must not be empty"
  • "Invalid uid"

Example:

json
{
  "success": false,
  "error": "cloud_access_code must not be empty"
}

Server Error (500 Internal Server Error) ​

Returns JSON with success: false and an error message describing the failure.

Example:

json
{
  "success": false,
  "error": "internal server error"
}

Unauthorized (401) ​

Returns plain text (not JSON):

NOT_LOGGED_IN

Behavior ​

  • The endpoint only schedules the reboot; it does not immediately reboot the device.
  • Ensure the caller has the objects scope; otherwise the request will be rejected.

Notes ​

  • Use POST with a JSON body — Content-Type: application/json is required.
  • All error responses for 400 and 500 return JSON with an error field. 401 is plain text NOT_LOGGED_IN.
